Лабораторная работа №5 "ИНСТРУМЕНТИРОВАНИЕ КОДА"
Задание 1. Аннотация Invoke

Программа демонстрирует механизм пользовательских аннотаций и их обработку через Reflection API. Позволяет создавать аннотированные методы, автоматически находить и вызывать их с помощью специального обработчика.

Основные компоненты системы

1. Аннотация @Invoke
   * Назначение: помечает методы, подлежащие автоматическому вызову.
   * Характеристики:
     * применяется только к методам (@Target(ElementType.METHOD));
     * доступна во время исполнения (@Retention(RetentionPolicy.RUNTIME));
     * не имеет параметров.

2. Класс SampleClass
   * Назначение: демонстрирует использование аннотации @Invoke.
   * Свойства: содержит три метода разного типа.
   * Методы:
     * simpleMethod() — простой метод без аннотации;
     * invokedMethod() — помечен @Invoke, вызывается автоматически;
     * anotherMethod() — метод без аннотации.

3. Класс Invoker
   * Назначение: обработчик аннотированных методов.
   * Функционал:
     * сканирует объект на наличие методов с @Invoke;
     * автоматически вызывает найденные методы через Reflection API;
     * обрабатывает возможные исключения при вызове.
   * Ключевые методы: invokeMethods(Object obj).

4. Класс Main
   * Назначение: точка входа в приложение, демонстрация работы системы.
   * Функционал:
     * создаёт экземпляр SampleClass;
     * использует Invoker для автоматического вызова методов с @Invoke;
     * демонстрирует явный вызов метода без аннотации;
     * обеспечивает дружественный интерфейс с пользователем.
